2.1 观察下表EMP (见题目图片1) 下列能够查询工号EMPNO在7300——7500之间的员工的姓名的SQL语句是？(请选出两项) （10 分）

图1


SELECT ENAME FROM EMP WHERE EMPNO IN(7300,7500);

源码1 


SELECT ENAME FROM EMP WHERE EMPNO BETWEEN 7300 AND 7500;

源码1 


SELECT ENAME FROM EMP WHERE EMPNO>7300 OR EMPNO<7500;

源码1 


SELECT ENAME FROM EMP WHERE EMPNO>7300 AND EMPNO<7500;

源码1

我的答案： (错误) 查看正确答案 正确答案：B,D
本题解析：
between and 的作用和> and <的作用相同

2.2 观察下表（EMP） (见题目图片1) 经理要求你查询出所有员工的姓名和工资，查询结果按工资降序排序，相同工资的按姓名升序排序，能够完成该需求的是？(请选出三项) （10 分）

图1


SELECT ENAME,SAL FROM EMP ORDER BY 2 DESC,ENAME ASC;

源码1 


SELECT ENAME,SAL FROM EMP ORDER BY SAL DESC,ENAME ASC;

源码1 


SELECT ENAME,SAL FROM EMP ORDER BY SAL,ENAME ASC;

源码1 


SELECT ENAME,SAL FROM EMP ORDER BY SAL DESC,ENAME;

源码1

我的答案： (错误) 查看正确答案 正确答案：A,B,D
本题解析：
oracle数据库排序方式默认为升序（ASC可以省略），要求降序是必须要在排序列后加desc； oracle数据库允许多重排序。 在这里最有疑问的可能就是A选项了，明明第一个排序的基准是2，而不时题目要求的以工资降序排序，为什么A选项正确呢？我们知道工资的数据类型是number类型，排序中2也是number数据类型，所以数据库系统就自作主张地把2当成sal来用了，当然这样是很合理的。

2.3 观察下表（EMP） (见题目图片1) 经理要求查询出不在20或30号部门的员工姓名及所在部门，下列能够完成该需求的是？(请选出两项) （10 分）

图1


SELECT ENAME,DEPTNO FROM EMP WHERE DEPTNO NOT IN(20,30);

源码1 


SELECT ENAME,DEPTNO FROM EMP WHERE DEPTNO <>20 OR DEPTNO <>30;

源码1 


SELECT ENAME,DEPTNO FROM EMP WHERE DEPTNO NOT =(20,30);

源码1 


SELECT ENAME,DEPTNO FROM EMP WHERE DEPTNO <>20 AND DEPTNO <>30;

源码1

我的答案： (错误) 查看正确答案 正确答案：A,D
本题解析：
查询某列的值不在某些值内最佳的方案就是用not in（不被包含的值）
